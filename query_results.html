Query results¶

Python code might be used to perform some operations. For instance, the following snippet loads libraries and SPARQL queries, and sends them to RDFox to be answered against the pre-converted data:

from pathlib import Path
import pandas as pd
from probs_ontology.runner.probs_runner import probs_query_data

queries = {
    p.stem: p.read_text()
    for p in Path("queries").glob("*.rq")
}

results = probs_query_data("../data/probs_data.nt.gz", queries)

Retrieve original data points¶

First, let’s check we can retrieve the original data points consistently. From the BGS Minerals Yearbook:

PREFIX : <https://ukfires.org/probs/ontology/>
PREFIX ufpc: <https://ukfires.org/probs/ontology/prodcom/>
PREFIX time: <http://www.w3.org/2006/time#>

SELECT ?Object ?Year ?Value
WHERE {
  ?Observation :partOfDataset ufpc:BGSMineralsYearbook2015 ;
               :objectDefinedBy [ :objectName ?Object ] ;
               :hasTimePeriod [ time:year ?Year ] ;
               :measurement ?Value .
  FILTER( STRENDS(?Object, "in BGS") )
}
ORDER BY ?Object ?Year

tidydf(results["original_bgs"])

Inferred observations¶

Now let’s query for all the observations that would be relevant to modelling production of two object types, CrushedStone and SandAndGravel, in all years that are available.

Fig. 2 All the observations relevant to Aggregates components (CrushedStone and SandAndGravel).¶

Here is the SPARQL query:

PREFIX : <https://ukfires.org/probs/ontology/>
PREFIX gn: <http://www.geonames.org/ontology#>
PREFIX time: <http://www.w3.org/2006/time#>

SELECT ?Observation ?Object ?Year ?Bound ?Value
WHERE {
  ?Observation :objectDefinedBy [ :objectName ?Object ] ;
               :hasRegion [ gn:name "United Kingdom" ] ;
               :hasRole :SoldProduction ;
               :hasTimePeriod [ time:year ?Year ] ;
               :bound ?Bound ;
               :measurement ?Value .
  FILTER( ?Object = "Crushed stone" || ?Object = "Sand & Gravel" )
  FILTER( ?Year = "2014"^^xsd:gYear || ?Year = "2018"^^xsd:gYear )
}
ORDER BY ?Object ?Bound ?Year

And the results:

tidydf(results["object_observations"])

We can see where these values have come from:

PREFIX : <https://ukfires.org/probs/ontology/>
PREFIX prov: <http://www.w3.org/ns/prov#>
PREFIX gn: <http://www.geonames.org/ontology#>
PREFIX time: <http://www.w3.org/2006/time#>

SELECT DISTINCT ?Observation ?WDFObject ?WDFValue
WHERE {
  ?Observation :objectDefinedBy [ :objectName ?Object ] ;
               :hasRegion [ gn:name "United Kingdom" ] ;
               :hasRole :SoldProduction ;
               :hasTimePeriod [ time:year ?Year ] ;
               prov:wasDerivedFrom ?WDF .

  FILTER( ?Object = "Crushed stone" || ?Object = "Sand & Gravel" )
  FILTER( ?Year = "2014"^^xsd:gYear || ?Year = "2018"^^xsd:gYear )

  ?WDF a :DirectObservation ;
         :objectDirectlyDefinedBy [ :objectName ?WDFObject ] ;
         :measurement ?WDFValue
}
ORDER BY ?Observation ?WDFObject

This results in:

df = tidydf(results["prov"])
df

It does indeed add up to the values shown above:

df.groupby("Observation", as_index=False)["WDFValue"].sum()

Further aggregation¶

We can further query for the aggregates observations of Aggregates:

PREFIX : <https://ukfires.org/probs/ontology/>
PREFIX gn: <http://www.geonames.org/ontology#>
PREFIX time: <http://www.w3.org/2006/time#>

SELECT ?Year ?Bound ?Value
WHERE {
  ?Observation :objectDefinedBy [ :objectName ?Object ] ;
               :hasRegion [ gn:name "United Kingdom" ] ;
               :hasRole :SoldProduction ;
               :hasTimePeriod [ time:year ?Year ] ;
               :bound ?Bound ;
               :measurement ?Value .
  FILTER( ?Object = "Aggregates" )
  FILTER( ?Year = "2014"^^xsd:gYear || ?Year = "2018"^^xsd:gYear )
}
ORDER BY ?Bound ?Year

This results in:

tidydf(results["object_observations_aggregates"])
