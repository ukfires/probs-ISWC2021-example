Object definitions¶

As before, load the query results:

from example_data import results

Here are just a few example object types which we might be interested in modelling.

Object: Aggregates

Object: CrushedStone / Crushed stone

Parent: Aggregates

Object: SandAndGravel / Sand & Gravel

Parent: Aggregates

Ending sub objects of “Aggregates”

Fig. 5 Aggregates Hierarchy.¶

Note that these are “new” objects that we have created because they are interesting for our analysis. We also call them Reference Objects.

We can retrieve them using the following query:

PREFIX : <https://ukfires.org/probs/ontology/>

SELECT DISTINCT ?Object
WHERE {
    ?Object a :ReferenceObject .
}

results["object_definitions"]
Object
0 sys:CrushedStone
1 sys:Aggregates
2 sys:SandAndGravel

And we can see the components of Aggregates with this query:

PREFIX : <https://ukfires.org/probs/ontology/>
PREFIX ufsys: <http://ukfires.org/probs/system/>

SELECT DISTINCT ?Object
WHERE {
    ufsys:Aggregates :objectComposedOf ?Object .
}

results["object_definitions_composition"]
Object
0 sys:CrushedStone
1 sys:SandAndGravel
